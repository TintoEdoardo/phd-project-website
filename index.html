    The vision

    For a more comprehensive exposition, consult the following white paper.

    The key traits of the envisioned Compute Continuum are the following.

    • Interactions over Web-level protocols
    • Edge and Cloud as a seamless, continuous, infrastructure
    • Opportunistic and live migration of application components
      In response to changes in the physical world

    Project's roadmap

    The construction of a runtime infrastructure for the Compute Continuum:

    • Construction of a node

      • Supporting live migration for Wasm computations:
        Migrating compiled module [GitHub]
      • Three deployment scenarios

        (1) Bare-metal, without an OS

        (2) On top of a RTOS

        (3) Leveraging a general-purpose OS

    • Construction of a cluster

      • Building A two-level orchestration strategy
        Application-level orchestration of migrating computations [GitHub]

        In collaboration with Prof. Alessandro Papadopoulos @ MDU

      • Enabling live migration in Oakestra
        New commands for Oakestra [GitHub]

        In collaboration with Giovanni Bartolomeo @ TUM

    • Benchmarking

      • Benchmark selection or realization
      • Experimental evaluation over the infrastructure

    Scenarios

    Several application scenarios might benefit from this runtime infrastructure.
    • High-Performance Computing
      To leverage an increasingly vast pool of computing resources, both in Cloud and Edge, for High-Performance Computing (HPC) applications
    • Real-Time Systems
      Live migration improves adaptability and reconfigurability in response to changes in the physical world
    • Energy Harvesting Systems
      Migrating computations in the presence of low-energy phases, aiming for higher system utilization
